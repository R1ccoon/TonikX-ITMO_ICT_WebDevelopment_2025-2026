    Доска домашних заданий¶

    Современная веб-система для управления домашними заданиями в учебных заведениях, построенная на Django с использованием Bootstrap 5.

    Возможности¶

    Для студентов:¶

    • Просмотр всех доступных заданий
    • Поиск и фильтрация заданий по предметам и статусу
    • Сдача заданий в текстовом виде
    • Просмотр своих сдач и оценок
    • Комментарии преподавателей к оценкам

    Для преподавателей:¶

    • Создание и управление заданиями
    • Оценка работ студентов
    • Просмотр статистики по предметам и студентам
    • Поиск и фильтрация оценок
    • Обратная связь через комментарии

    Для администраторов:¶

    • Управление пользователями
    • Полная статистика системы
    • Настройка предметов и ролей
    • Доступ к админ-панели Django

    Технологии¶

    • Backend: Django 5.2.7
    • Frontend: Bootstrap 5.3.0, HTML5, CSS3, JavaScript
    • База данных: SQLite (для разработки)
    • Иконки: Bootstrap Icons
    • Язык: Python 3.12+

    Требования¶

    • Python 3.12+
    • Django 5.2.7
    • pip (менеджер пакетов Python)

    Установка и запуск¶

    1. Клонирование репозитория¶

    git clone <repository-url>
cd homework_board

    2. Создание виртуального окружения¶

    python -m venv homework-env
source homework-env/bin/activate  # Linux/Mac
# или
homework-env\Scripts\activate.bat  # Windows

    3. Установка зависимостей¶

    pip install django

    4. Применение миграций¶

    python manage.py migrate

    5. Создание суперпользователя¶

    python manage.py createsuperuser

    6. Заполнение тестовыми данными¶

    python populate_data.py

    7. Запуск сервера¶

    python manage.py runserver

    8. Открытие в браузере¶

    Перейдите по адресу: http://127.0.0.1:8000/

    Особенности дизайна¶

    • Адаптивный дизайн для всех устройств
    • Современный интерфейс с Bootstrap 5
    • Интуитивная навигация с иконками
    • Цветовая индикация статусов и оценок
    • Анимации и переходы для улучшения UX
    • Модальные окна для детальной информации

    Функциональность¶

    Поиск и фильтрация:¶

    • Поиск заданий по названию и описанию
    • Фильтрация по предметам и статусу
    • Поиск оценок по студентам и заданиям

    Пагинация:¶

    • Удобная навигация по страницам
    • Настраиваемое количество элементов на странице
    • Сохранение параметров поиска при переходах

    Роли пользователей:¶

    • Студент: просмотр заданий, сдача работ, просмотр оценок
    • Преподаватель: создание заданий, оценка работ, статистика
    • Администратор: полный доступ ко всем функциям

    Статистика¶

    Система предоставляет подробную статистику: - Общее количество заданий, сдач и оценок - Статистика по предметам - Топ студентов по среднему баллу - Распределение оценок по категориям

    Настройка¶

    Изменение количества элементов на странице:¶

    В файле settings.py: 

    PAGINATE_BY = 10  # Количество элементов на странице

    Добавление новых предметов:¶

    Через админ-панель Django или программно: 

    Subject.objects.create(name="Новый предмет", description="Описание")

    Развертывание¶

    Для продакшена рекомендуется:¶

    1. Использовать PostgreSQL вместо SQLite
    2. Настроить статические файлы с помощью WhiteNoise
    3. Использовать Gunicorn как WSGI сервер
    4. Настроить Nginx как reverse proxy
    5. Использовать переменные окружения для настроек

    Пример настройки для продакшена:¶

    # settings.py
import os

DEBUG = False
ALLOWED_HOSTS = ['yourdomain.com']

DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql',
        'NAME': os.environ.get('DB_NAME'),
        'USER': os.environ.get('DB_USER'),
        'PASSWORD': os.environ.get('DB_PASSWORD'),
        'HOST': os.environ.get('DB_HOST'),
        'PORT': os.environ.get('DB_PORT'),
    }
}
