# Система управления автовладельцами

Веб-приложение на Django для управления информацией об автовладельцах, их автомобилях и водительских удостоверениях.

## Описание проекта

Этот проект представляет собой учебное веб-приложение, демонстрирующее основные возможности Django:
- Создание моделей данных
- Работа с формами
- Админ-панель Django
- Система аутентификации
- Шаблоны и представления

## Структура проекта

- **django_project_tonikx** - основной проект Django
- **project_first_app** - приложение для управления автовладельцами
- **templates** - HTML шаблоны
- **tutorial-env** - виртуальное окружение Python

## Модели данных

### Owner (Владелец)
- `first_name` - Имя
- `last_name` - Фамилия  
- `birth_date` - Дата рождения

### Car (Автомобиль)
- `brand` - Марка
- `model` - Модель
- `color` - Цвет
- `state_number` - Государственный номер

### Ownership (Владение)
- `owner` - Владелец (внешний ключ)
- `car` - Автомобиль (внешний ключ)
- `start_date` - Дата начала владения
- `end_date` - Дата окончания владения (может быть пустой)

### DriverLicense (Водительское удостоверение)
- `owner` - Владелец (внешний ключ)
- `license_number` - Номер удостоверения
- `license_type` - Тип удостоверения (A, B, C, D)
- `issue_date` - Дата выдачи

## Установка и запуск

### 1. Активация виртуального окружения
```bash
source tutorial-env/bin/activate
```

### 2. Применение миграций
```bash
python manage.py migrate
```

### 3. Создание суперпользователя
```bash
python manage.py createsuperuser
```

### 4. Заполнение тестовыми данными
```bash
python populate_data.py
```

### 5. Запуск сервера разработки
```bash
python manage.py runserver
```

### 6. Доступ к приложению
- **Главная страница**: http://127.0.0.1:8000/
- **Админ-панель**: http://127.0.0.1:8000/admin/
- **Информация о владельце**: http://127.0.0.1:8000/owner/1/

## Админ-панель

Для доступа к админ-панели используйте:
- **Логин**: admin
- **Пароль**: admin123

В админ-панели можно:

## Настройка

### Изменение настроек базы данных:
В файле `settings.py`:
```python
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.sqlite3',
        'NAME': BASE_DIR / 'db.sqlite3',
    }
}
```

### Добавление новых полей в модели:
1. Измените модель в `models.py`
2. Создайте миграцию: `python manage.py makemigrations`
3. Примените миграцию: `python manage.py migrate`

## Развертывание

### Для продакшена рекомендуется:
1. Использовать PostgreSQL вместо SQLite
2. Настроить статические файлы
3. Использовать Gunicorn как WSGI сервер
4. Настроить Nginx как reverse proxy
5. Использовать переменные окружения для настроек
